# Last Sunday — C# Walkthrough

This is an **algorithmic kata**: inputs are two `int`s (`year` and `month`, the latter 1-indexed per human convention) and the output is a `DateOnly` from the standard library — so there are no domain builders, no value types, and no collaborators to introduce. The reference lands as a single commit: `src/LastSunday/LastSunday.cs` exposes a static `Find(int year, int month)` method that builds the last day of the month via `DateTime.DaysInMonth`, computes `(lastDay.DayOfWeek - DayOfWeek.Sunday + 7) % 7` to get the number of days to walk back, and returns `lastDay.AddDays(-daysToWalkBack)`. `tests/LastSunday.Tests/LastSundayTests.cs` has one `[Fact]` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec.

**Inline literals — deliberate.** The `7` and the walk-back arithmetic stay inline rather than hiding behind `DaysInWeek = 7`. In F1 katas the function body fits on one screen and the literals *are* the rule — naming `DaysInWeek` would write "seven" twice without adding meaning. F3 and larger modes prefer named constants; F1 deliberately doesn't. The `+ 7) % 7` idiom guards against negative modulo when the last day is itself Sunday (distance zero), which is the `2013-03-31` / `2032-02-29` scenario pair.
