# Last Sunday — Python Walkthrough

This is an **algorithmic kata**: inputs are two `int`s (`year` and `month`, the latter 1-indexed per human convention) and the output is a standard-library `datetime.date` — so there are no domain builders, no value types, and no collaborators to introduce. The reference lands as a single commit: `src/last_sunday/last_sunday.py` defines `find(year: int, month: int) -> date` which uses `calendar.monthrange` to get the number of days in the month, constructs the last day, then walks back `(weekday - 6 + 7) % 7` days to Sunday. Python's `date.weekday()` returns Monday=0 through Sunday=6, so Sunday is `6` rather than C#'s / JavaScript's `0` — the modular arithmetic is identical, only the target-weekday integer changes. The package `__init__.py` re-exports `find` so tests import it as `from last_sunday import find`. `tests/test_last_sunday.py` has one function per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec.

**Inline literals — deliberate.** The `6`, the `7`, and the walk-back arithmetic stay inline rather than hiding behind `SUNDAY = 6` / `DAYS_IN_WEEK = 7`. In F1 katas the function body fits on one screen and the literals *are* the rule. F3 and larger modes prefer named constants; F1 deliberately doesn't. The `+ 7) % 7` idiom guards against negative modulo when the last day is itself Sunday (distance zero), which is the `2013-03-31` / `2032-02-29` scenario pair.
