# Balanced Brackets — TypeScript Walkthrough

This is an **algorithmic kata**: the input is a `string`, the output is a `boolean` — so there are no domain builders, no value types, and no collaborators to introduce. The reference lands as a single commit. `src/balancedBrackets.ts` exports one pure function: `isBalanced(input)`. The algorithm is a single-pass depth counter — increment on `[`, decrement on `]`, fail fast when the counter goes negative, and require the final counter to be zero. An array used as a stack would work too, but because the only bracket type is `[]` the stack never carries information the counter can't — a `number` is the honest representation. `tests/balancedBrackets.test.ts` has one `it()` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md), grouped under `describe('isBalanced', …)` to match the exported surface.

**Inline literals — deliberate.** Scenario literals like `'[[]]'` and `'[][]]['` sit inline in the tests rather than behind named constants. In F1 katas the inputs *are* the domain — extracting `const DEEPLY_NESTED = '[[[][]]]'` would write the string twice without adding meaning. F3 and larger modes prefer named constants and builders; F1 deliberately doesn't.
