# Rock Paper Scissors ‚Äî C# Walkthrough

This is an **algorithmic kata** with a small closed domain: two enums ‚Äî `Play { Rock, Paper, Scissors }` and `Outcome { Win, Lose, Draw }` ‚Äî and a pure function `Game.Decide(Play p1, Play p2) -> Outcome` that returns the result from player 1's perspective. There are no builders, no value types beyond the enums, and no collaborators ‚Äî the enums *are* the domain. The reference lands as a single commit across three source files (`Play.cs`, `Outcome.cs`, `Game.cs`) and one test file (`GameTests.cs`). `Game.Decide` short-circuits on equal plays as a draw, then delegates to a pattern-match on the three beat pairs ‚Äî `Rock‚ÜíScissors`, `Scissors‚ÜíPaper`, `Paper‚ÜíRock` ‚Äî returning `Win` if player 1's play beats player 2's and `Lose` otherwise. `tests/RockPaperScissors.Tests/GameTests.cs` enumerates all nine cases of the 3√ó3 matrix as discrete `[Fact]`s; no `[Theory]` ‚Äî for sibling parity with `fizz-buzz-whiz` and because nine named sentences read better than a data-driven table for a domain this thin.

**Typed enums over primitives ‚Äî deliberate.** F1 normally keeps inputs as primitives (the input to `IsLeapYear` is a raw `int`, the input to `FizzBuzzWhiz.Say` is a raw `int`). Here the domain has a closed set of three plays and three outcomes with exact names the business uses ‚Äî using `string` would invite `"Rock"` vs `"rock"` drift across call sites, and using `int` would lose the names entirely. The enum is the ubiquitous language, not infrastructure.
