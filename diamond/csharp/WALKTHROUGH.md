# Diamond — C# Walkthrough

This is an **algorithmic kata**: the input is a single character and the output is a multi-line ASCII string, so there are no domain builders, no value types, and no collaborators to introduce — the inputs and outputs *are* the domain. The reference lands as a single commit: `src/Diamond/Diamond.cs` exposes `Print(char)` which normalizes via `char.ToUpperInvariant`, rejects anything outside `A`–`Z` with `ArgumentException` (message `letter must be a single A-Z character`), then walks rows `0..2n` computing each row's offset as `min(r, 2n - r)` and building the row with a `StringBuilder`: `n - offset` leading spaces, the row letter, then — when `offset > 0` — `2 * offset - 1` inner spaces and a second copy of the letter. No trailing whitespace; lines joined with `\n` and no terminating newline. `tests/Diamond.Tests/DiamondTests.cs` has one `[Fact]` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec.

**Inline literals — deliberate.** The mirror-offset arithmetic (`2 * offset - 1`, `2 * n + 1`) is itself the rule — naming `InnerSpaceCount = 2 * offset - 1` just restates what the code already says. F3 and larger modes prefer named constants; F1 deliberately doesn't.
