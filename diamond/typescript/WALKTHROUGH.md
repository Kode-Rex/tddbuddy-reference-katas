# Diamond — TypeScript Walkthrough

This is an **algorithmic kata**: the input is a single character string and the output is a multi-line ASCII string, so there are no domain builders, no value types, and no collaborators to introduce — the inputs and outputs *are* the domain. The reference lands as a single commit: `src/diamond.ts` exports `print(letter)` which rejects non-single-character strings and anything outside `A`–`Z` / `a`–`z` with a plain `Error` (message `letter must be a single A-Z character` — byte-identical across languages; exception type is idiomatic per language), normalizes with `.toUpperCase()`, then walks rows `0..2n` computing each row's offset as `min(r, 2n - r)` and building the row via `String.prototype.repeat`: `n - offset` leading spaces, the row letter, then — when `offset > 0` — `2 * offset - 1` inner spaces and a second copy of the letter. No trailing whitespace; lines joined with `\n` and no terminating newline. `tests/diamond.test.ts` has one `it()` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec.

**Inline literals — deliberate.** The mirror-offset arithmetic (`2 * offset - 1`, `2 * n + 1`) is itself the rule — naming `innerSpaceCount = 2 * offset - 1` just restates what the code already says. F3 and larger modes prefer named constants; F1 deliberately doesn't.
