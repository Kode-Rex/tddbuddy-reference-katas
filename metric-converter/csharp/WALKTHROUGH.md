# Metric Converter ‚Äî C# Walkthrough

This is an **algorithmic kata** with a small closed domain: one enum ‚Äî `Unit { Kilometers, Miles, Celsius, Fahrenheit, Kilograms, Pounds, Liters, UsGallons, UkGallons }` ‚Äî and a pure function `Converter.Convert(double value, Unit from, Unit to) -> double` that dispatches on the `(from, to)` pair. There are no builders, no value types beyond the enum, and no collaborators ‚Äî the enum *is* the domain. The reference lands as a single commit across two source files (`Unit.cs`, `Converter.cs`) and one test file (`ConverterTests.cs`). `Converter.Convert` is a single pattern-match over the five supported pairs ‚Äî `Kilometers‚ÜíMiles`, `Celsius‚ÜíFahrenheit`, `Kilograms‚ÜíPounds`, `Liters‚ÜíUsGallons`, `Liters‚ÜíUkGallons` ‚Äî with every unsupported pair (e.g. `Kilometers‚ÜíFahrenheit`) falling through to an `ArgumentException`. Conversion constants appear inline; the kata spec names them by value (`0.621371`, `0.45359237`, `3.785411784`, `4.54609`) and that is the rule ‚Äî naming them `KilometersPerMile` would restate the literal twice without adding meaning. `tests/MetricConverter.Tests/ConverterTests.cs` enumerates one `[Fact]` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); approximate assertions use `FluentAssertions`'s `BeApproximately` with a tolerance small enough to catch a formula error but loose enough to absorb binary-float rounding.

**Typed enum over primitives ‚Äî deliberate.** F1 normally keeps inputs as primitives (the input to `IsLeapYear` is a raw `int`), but here the `from`/`to` pair is a closed set with business-named members. Using `string` would invite `"km"` vs `"Kilometers"` drift across call sites, and a bag of overloaded methods (`KilometersToMiles`, `CelsiusToFahrenheit`, ‚Ä¶) would spread one concept over five signatures. One entry point with typed units keeps the spec's sentence shape ‚Äî "convert *value* from *unit* to *unit*" ‚Äî and lets the switch close exhaustively over the supported pairs.
