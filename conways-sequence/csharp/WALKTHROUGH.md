# Conway's Sequence — C# Walkthrough

This is an **algorithmic kata**: inputs and outputs are digit strings, so there are no domain builders, no value types, and no collaborators to introduce — the inputs and outputs *are* the domain. The reference lands as a single commit: `src/ConwaysSequence/ConwaysSequence.cs` exposes `NextTerm(string)` — a single left-to-right pass that accumulates each maximal run length and emits `<length><digit>` into a `StringBuilder` — and `LookAndSay(string, int)` which applies `NextTerm` `iterations` times, returning the seed unchanged when `iterations` is zero and throwing `ArgumentOutOfRangeException` when it is negative. `tests/ConwaysSequence.Tests/ConwaysSequenceTests.cs` has one `[Fact]` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec. The run-length integer is formatted with `CultureInfo.InvariantCulture` so double-digit runs (`"1111111111"` → `"101"`) are locale-independent.

**Inline literals — deliberate.** The nested-loop pattern fits on one screen, and there are no magic numbers begging to be named — `runLength` and `digit` already carry the meaning. F3 and larger modes prefer named constants; F1 deliberately doesn't.
