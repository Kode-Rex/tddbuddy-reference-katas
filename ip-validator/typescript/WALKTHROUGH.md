# IP Validator — TypeScript Walkthrough

This is an **algorithmic kata**: the input is a `string` and the output is a `boolean`, so there are no domain builders, no value types, and no collaborators to introduce — the inputs and outputs *are* the domain. The reference lands as a single commit: `src/ipValidator.ts` exports `isValid(input: string): boolean`. The function splits on `.`, rejects anything that isn't exactly four segments, and then walks each octet through a local `parseOctet` that returns `null` on empty octets, leading zeros, non-digits, or values above 255 — reusing the overflow check as the >255 guard so there's a single running total per octet. A final check rejects a last octet of `0` or `255` (network and broadcast). `tests/ipValidator.test.ts` has one `it()` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec.

**No regex — deliberate.** The TDD Buddy prompt forbids it, and hand-rolling the digit scan makes the leading-zero and negative-sign rejections fall out naturally from the character loop (a `-` is simply not a digit). `Number(octet)` or `parseInt` would variously accept `'007'`, `' 1 '`, or `'1e2'` — none of which are valid here. **Inline literals — deliberate.** The bounds `0`, `255`, and the octet count `4` are kept inline rather than extracted as named constants. In F1 katas the function body fits on one screen and the literals *are* the rule.
