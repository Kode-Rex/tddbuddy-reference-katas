# URL Shortener — C# Walkthrough

This is an **F1 kata with state**. The SUT is a `Shortener` class holding three dictionaries (`_longToCode`, `_codeToLong`, `_visits`) and a counter; tests instantiate it fresh per scenario and exercise `Shorten`, `Translate`, and `Statistics` directly. No builder is needed — a bare `new Shortener()` is the only setup, and the 14 scenarios in [`../SCENARIOS.md`](../SCENARIOS.md) read as direct calls against that instance. The output of `Statistics` is a `record UrlStatistics(string ShortUrl, string LongUrl, int Visits)` so tests assert the whole triple with `Should().Be(new UrlStatistics(...))` instead of three separate property assertions. `src/UrlShortener/UrlShortener.cs` holds the record and the class in one file; everything fits on one screen.

**Short-code scheme: base36 sequential — deliberate.** The spec's `abc1234` example suggests a hash; we chose a 0-indexed counter rendered as base36 (`0`, `1`, …, `9`, `a`, …, `z`, `10`, …). This makes the scenarios spec-able ("the first URL shortens to `.../0`, the eleventh to `.../a`") without forcing the reader to pre-compute a hash digest. Collision handling disappears. **`ShortUrlBase` is a named constant — deliberate.** Unlike most F1 katas where inline literals win, `https://short.url/` appears in four places (shorten return, translate return, statistics shape, short-URL prefix check) and *is* the public contract, so the name earns its keep. **`ArgumentException` for unknown URLs — idiomatic.** TS uses `Error`, Python uses `ValueError`; the message string `Unknown URL: <url>` is byte-identical across the three languages so a cross-language reader can grep for the exact phrase.
