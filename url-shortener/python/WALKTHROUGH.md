# URL Shortener — Python Walkthrough

This is an **F1 kata with state**. The SUT is a `Shortener` class holding three dicts (`_long_to_code`, `_code_to_long`, `_visits`) and a counter; tests instantiate it fresh per scenario and exercise `shorten`, `translate`, and `statistics` directly. No builder is needed — `Shortener()` is the only setup, and the 14 scenarios in [`../SCENARIOS.md`](../SCENARIOS.md) read as direct method calls. The output of `statistics` is a `UrlStatistics = @dataclass(frozen=True)` with `short_url`, `long_url`, `visits` fields so tests compare whole objects with `==`.

**Short-code scheme: base36 sequential — deliberate.** A small hand-rolled `_to_base36(value)` converts the 0-indexed counter (`0`, `1`, …, `9`, `a`, …, `z`, `10`). Python's stdlib has no direct int-to-base36 helper, but the loop is six lines and keeps the intent visible (the spec says "first URL gets `0`, eleventh gets `a`" — that's the code, not a magic number). **`SHORT_URL_BASE` is a module-level named constant — deliberate.** Unlike most F1 katas where inline literals are the rule, the base URL *is* the public contract and appears in four places (shorten return, translate return, statistics shape, short-URL prefix check). **`ValueError` for unknown URLs — idiomatic.** C# uses `ArgumentException`, TS uses `Error`; the message string `Unknown URL: <url>` is byte-identical across all three languages.
