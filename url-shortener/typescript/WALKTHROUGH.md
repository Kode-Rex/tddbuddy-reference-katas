# URL Shortener — TypeScript Walkthrough

This is an **F1 kata with state**. The SUT is a `Shortener` class backed by three `Map`s (long-to-code, code-to-long, code-to-visits) and a counter; tests instantiate it fresh per scenario and exercise `shorten`, `translate`, and `statistics` directly. No builder is needed — `new Shortener()` is the only setup, and the 14 scenarios in [`../SCENARIOS.md`](../SCENARIOS.md) read as direct method calls. The output of `statistics` is a `UrlStatistics` interface (three fields: `shortUrl`, `longUrl`, `visits`) so tests assert whole objects with `toEqual({...})`.

**Short-code scheme: base36 sequential — deliberate.** The counter-to-code conversion collapses to `value.toString(36)`, which is exactly what we want: `0`, `1`, …, `9`, `a`, …, `z`, `10`. The scenarios can name expected codes directly without pre-computing a hash. **`SHORT_URL_BASE` is a module-level named constant — deliberate.** Unlike most F1 katas where inline literals are the rule, the base URL *is* the public contract and appears in five places (shorten return, translate return, statistics shape, short-URL prefix detection, `slice` offset). **`Error` for unknown URLs — idiomatic.** C# uses `ArgumentException`, Python uses `ValueError`; the message string `Unknown URL: <url>` is byte-identical across all three languages.
