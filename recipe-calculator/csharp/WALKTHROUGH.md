# Recipe Calculator — C# Walkthrough

This is an **algorithmic kata**: the input is a `IReadOnlyDictionary<string, double>` and a scale factor, the output is a new `IReadOnlyDictionary<string, double>`, so there are no domain builders, no value types, and no collaborators to introduce — the inputs and outputs *are* the domain. The reference lands as a single commit: `src/RecipeCalculator/RecipeCalculator.cs` is a static `Scale(recipe, factor)` method that rejects negative factors with `ArgumentOutOfRangeException` and otherwise walks the entries multiplying each quantity by `factor`. `tests/RecipeCalculator.Tests/RecipeCalculatorTests.cs` has one `[Fact]` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec. The kata's bonus tracks (unit conversion, dietary restrictions, substitutions, shopping lists, nutrition) are intentionally out of scope — unit conversion is covered by the sibling [`metric-converter`](../../metric-converter/) kata, and scaling alone is sufficient to demonstrate the F1 shape.

**Inline literals — deliberate.** The guard `factor < 0` is kept inline rather than extracted as a named predicate. In F1 katas the function body fits on one screen and the guard *is* the rule — naming `IsNegative(factor)` would restate "less than zero" twice without adding meaning. F3 and larger modes prefer named predicates; F1 deliberately doesn't.
