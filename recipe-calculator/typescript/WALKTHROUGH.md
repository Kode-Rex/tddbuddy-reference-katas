# Recipe Calculator — TypeScript Walkthrough

This is an **algorithmic kata**: the input is a `Record<string, number>` and a scale factor, the output is a new `Record<string, number>`, so there are no domain builders, no value types, and no collaborators to introduce — the inputs and outputs *are* the domain. The reference lands as a single commit: `src/recipeCalculator.ts` exports a `scale(recipe, factor)` function that throws `RangeError` for negative factors and otherwise walks `Object.entries` multiplying each quantity by `factor`. `tests/recipeCalculator.test.ts` has one `it()` per scenario in [`../SCENARIOS.md`](../SCENARIOS.md); each test name reads as a sentence from that spec. The kata's bonus tracks (unit conversion, dietary restrictions, substitutions, shopping lists, nutrition) are intentionally out of scope — unit conversion is covered by the sibling [`metric-converter`](../../metric-converter/) kata, and scaling alone is sufficient to demonstrate the F1 shape.

**Inline literals — deliberate.** The guard `factor < 0` is kept inline rather than extracted as a named predicate. In F1 katas the function body fits on one screen and the guard *is* the rule. F3 and larger modes prefer named predicates; F1 deliberately doesn't.
